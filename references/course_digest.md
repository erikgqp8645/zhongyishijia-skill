# 中医世家 (ZYSJ) — 知识库总览

## 数据来源

本知识库来源于 **中医世家** 网站 (zysj.com.cn) 2012-2014 年的完整数据集, 包含 678 本古医书 CHM 文件 + 两个数据库快照 (2012 MSSQL / 2014 MySQL)。

## 数据来源溯源

- **古医书 (中医著作)**: 来自 `chm.rar` 解包, 包含 556 本古医书如《黄帝内经》《伤寒论》《本草纲目》《针灸大成》《景岳全书》《脉经》《难经》《温病条辨》等
- **中医教材**: 来自 chm 教材目录, 15 本现代中医教材
- **使用手册**: 41 本工具书, 包括 9 卷《中医词典》、法规、护理手册等
- **西医备考**: 65 本现代医学参考书
- **中药字典**: 来自 `zysjmssqlbak` MySQL 数据库的 `zysjyj` 表 (70,350 味), 按拼音首字母分 24 个文件
- **临床理论**: 来自 `zysjllsj` 表 (166,423 条), 按 TypeID 分类, 共 521 个分类
- **综合数据**: 来自 `zysjzhsj` 表 (80,809 条), 9 个 TypeID

## 蒸馏方法

本 Skill 通过 lineage-skill 工具生成, 采用纯本地 extractive 模式 (不依赖 LLM):

1. **CHM → Markdown**: 用 `hh.exe -decompile` 解包 + BeautifulSoup 解析 HHC 目录 + 提取 HTML 正文 (去导航表/页脚)
2. **SQLite → Markdown**: 按类别(TypeID/拼音)分文件, 每条记录一个章节, 清洗 HTML 实体和 BBCode 标签
3. **章节化**: 每本书/每类药/每条理论都按 H1/H2/H3 分级, 便于 AI 检索和引用
4. **元数据保留**: 保留 TypeID、ID、拼音、来源出处等结构化字段, 作为引用锚点

## 角色定位

- **expert**: 用于精确 Q&A 和引用原书 (回答如"人参归经？", 引用《本草纲目》)
- **mentor**: 用于学习辅导和路径推荐 (回答如"我想学中医从哪开始？")

## 边界

- 本知识库为**学习参考资料**, 不替代专业医生诊疗
- 部分古医书有现代医学观点, 仅供参考
- 数据库内容来自 2010-2014 年, 部分观点可能过时
